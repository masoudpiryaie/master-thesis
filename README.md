```markdown # A Two-Stage Reciprocal Recommender System for Profile-Based Compatibility Modeling This repository contains the code and notebook for an MSc dissertation project on **profile-based compatibility modeling** using a **two-stage reciprocal recommender system**. The project investigates how compatibility between users can be modeled using **public profile data**, especially in settings where direct behavioural interaction logs such as swipes, replies, or confirmed matches are not available. The implementation is based on the **OkCupid Profiles** dataset and focuses on: - data preprocessing - pairwise feature engineering - synthetic compatibility label construction - baseline models - two-stage candidate generation and reciprocal ranking - offline evaluation --- ## Project Overview Traditional recommender systems usually focus on **user-item recommendation**, where the goal is to predict how relevant an item is for a user. In contrast, this project studies **people-to-people recommendation**, where both sides of a recommendation matter. In online dating-style recommendation, a useful match should not only be relevant to one user, but should also be **mutually plausible** for both users. For this reason, this project implements a **two-stage reciprocal recommender system**: ### Stage 1: Candidate Generation A lightweight retrieval step identifies a smaller set of plausible candidate profiles. ### Stage 2: Reciprocal Ranking The retrieved candidates are re-ranked using compatibility features and a reciprocal scoring mechanism. Because the dataset does not contain real interaction outcomes, the project uses **synthetic but interpretable compatibility labels** derived from profile-level rules. --- ## Research Goal The main research question of this project is: > How can a two-stage reciprocal compatibility-based recommender system be constructed and experimentally evaluated using profile-level data in the absence of explicit interaction logs? --- ## Repository Structure ```bash . ├── README.md ├── okcupid_colab_notebook.ipynb ├── run_experiments.py └── src/ ├── __init__.py ├── baselines.py ├── evaluation.py ├── features.py ├── labels.py ├── preprocessing.py └── twostage_model.py ``` --- ## Files Description ### `run_experiments.py` Main runner script for the full experiment pipeline.
